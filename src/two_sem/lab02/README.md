## Реализованный класс коллекции: TeamCollection
Коллекция для хранения и управления объектами Team из ЛР-1. Реализует контейнер для группы спортивных команд с возможностью добавления, удаления, поиска, сортировки и фильтрации.

### Краткое описание класса
TeamCollection — это контейнерный класс для управления коллекцией спортивных команд. Внутри хранит список объектов Team.

### Основные возможности:

**Хранение** — список объектов Team

**Управление** — добавление, удаление, доступ по индексу

**Поиск** — по названию команды, дивизиону, статусу здоровья

**Сортировка** — по названию, очкам, производительности, мотивации

**Фильтрация** — получение подколлекций (активные, здоровые, высокопроизводительные)

**Итерация** — поддержка цикла for

**Защита** — проверка типа, запрет дубликатов по имени

## Методы коллекции
### Базовые операции:

**add(team)** — добавить команду

**remove(team)** — удалить команду

**remove_at(index)** — удалить по индексу

**get_all()** — получить список всех команд

## Поиск:

**find_by_name(name)** — поиск по названию команды

**find_by_division(division)** — поиск по дивизиону

**find_by_health_status(status)** — поиск по статусу здоровья

**find_active()** — поиск активных команд

### Магические методы:

__len__ — получение количества команд

__iter__ — итерация по коллекции

__getitem__ — доступ по индексу

__contains__ — проверка наличия команды

### Сортировка:

sort_by_name() — сортировка по названию

sort_by_points() — сортировка по очкам

sort_by_performance() — сортировка по производительности

sort_by_morale() — сортировка по мотивации

sort(key) — универсальная сортировка с пользовательским ключом

### Фильтрация (возвращают новую коллекцию TeamCollection):

get_active_teams() — только активные команды

get_healthy_teams() — только здоровые команды

get_high_performance_teams(threshold) — команды с высокой производительностью

get_teams_in_division(division) — команды из указанного дивизиона
