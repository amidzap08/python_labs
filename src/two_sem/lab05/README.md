## 1. Расширенная коллекция FunctionalTeamCollection

Коллекция реализована как автономный класс с методами функционального программирования. Поддерживает цепочки вызовов для последовательной обработки данных.

### Методы:

**`add(team)`** — добавление команды в коллекцию.

**`sort_by(key_func)`** — сортировка коллекции на месте по переданной ключевой функции. Возвращает `self` для поддержки цепочек вызовов.

**`filter_by(predicate)`** — создает новую коллекцию из команд, удовлетворяющих предикату. Исходная коллекция не изменяется. Возвращает новый объект `FunctionalTeamCollection`.

**`apply(func)`** — применяет переданную функцию ко всем командам в коллекции. Функция может модифицировать команду или просто обрабатывать её. Возвращает `self` для цепочек.

**`get_items()`** — возвращает копию списка всех команд для использования с внешними функциями (`map`, `filter`).

### Поддержка цепочек вызовов:

```python
result = (collection
    .filter_by(is_active)
    .sort_by(by_points)
    .apply(add_bonus))
```
Каждый метод (кроме filter_by) возвращает self, что позволяет объединять операции в цепочки. Метод filter_by создает новую коллекцию.

2. Стратегии сортировки
Реализовано 5 стратегий сортировки, каждая принимает объект команды и возвращает ключ для сортировки:

by_name(team) — сортировка по имени команды в алфавитном порядке (регистронезависимо). Возвращает имя в нижнем регистре.

by_points(team) — сортировка по количеству набранных очков по убыванию. Возвращает отрицательное значение очков.

by_bmi(team) — сортировка по индексу массы тела по возрастанию. Возвращает значение ИМТ.

by_performance(team) — сортировка по показателю производительности по убыванию. Возвращает отрицательную производительность.

by_division_and_name(team) — сортировка сначала по дивизиону, затем по имени. Возвращает кортеж (дивизион, имя).

3. Функции-фильтры
Реализовано 5 функций-предикатов для фильтрации команд:

is_active(team) — проверяет, активна ли команда. Возвращает True если team.is_active == True.

is_beginner(team) — проверяет принадлежность к дивизиону "beginner". Возвращает True если team.division == "beginner".

has_high_performance(team) — проверяет, превышает ли производительность порог 20. Возвращает True если team.performance_score() > 20.

is_professional(team) — проверяет, является ли команда профессиональной. Использует isinstance(team, ProfessionalTeam).

is_youth(team) — проверяет, является ли команда молодежной. Использует isinstance(team, YouthTeam).

4. Функции для map (преобразования)
Реализовано 4 функции для преобразования объектов команд в другие форматы:

get_name(team) — извлекает имя команды. Возвращает строку.

get_performance(team) — извлекает показатель производительности. Возвращает число с плавающей точкой.

get_team_summary(team) — создает краткую сводку по команде. Возвращает строку с именем, дивизионом, очками и производительностью.

get_team_dict(team) — преобразует команду в словарь с ключами: name, type, division, points, bmi, performance, active.

Пример использования:

```python
names = list(map(get_name, collection.get_items()))
dicts = list(map(get_team_dict, collection.get_items()))
```
5. Фабрика функций
Реализовано 2 фабричные функции, которые создают и возвращают новые функции-фильтры с заданными параметрами:

make_division_filter(division) — создает фильтр команд по указанному дивизиону. Принимает название дивизиона, возвращает функцию-предикат.

make_points_filter(min_points) — создает фильтр команд по минимальному количеству очков. Принимает пороговое значение, возвращает функцию-предикат.

Пример использования:

```python
beginner_filter = make_division_filter("beginner")
beginners = collection.filter_by(beginner_filter)
strong_filter = make_points_filter(150)
strong_teams = collection.filter_by(strong_filter)
```
6. Callable-объекты (паттерн Стратегия)
Реализовано 4 класса-стратегии, каждый из которых является callable-объектом (можно вызывать как функцию):

ActivateAllStrategy — стратегия активации всех команд. При вызове для каждой команды выполняет team.activate(), если команда неактивна. Не имеет параметров инициализации.

SetDivisionStrategy(target_division) — стратегия перевода команд в указанный дивизион. Принимает название целевого дивизиона. Для каждой команды вызывает team.set_division(target_division), если текущий дивизион отличается.

AddPointsStrategy(points) — стратегия добавления очков командам. Принимает количество очков для добавления. Для каждой команды вызывает team.add_points(points).

TeamInfoStrategy — стратегия вывода информации о команде. Не модифицирует команду, только выводит подробные данные: имя, тип, дивизион, очки, ИМТ, производительность, статус активности. Не имеет параметров.

Каждая стратегия реализует метод __call__(self, team), что позволяет использовать объект как функцию:

```python
strategy = AddPointsStrategy(50)
collection.apply(strategy)  # применяет +50 очков ко всем командам
```
7. Lambda-выражения
Lambda-выражения используются в демонстрации для простых операций, которые не требуют переиспользования:

Сортировка по очкам через lambda:

```python
collection.sort_by(lambda t: -t.total_points)
Сортировка по ИМТ через lambda:
```
```python
collection.sort_by(lambda t: t.avg_bmi())
Фильтрация с составным условием через lambda:
```
```python
collection.filter_by(lambda t: t.is_active and t.total_points > 100)
```
Для переиспользуемой логики применяются именованные функции с docstring.

10. Выводы
Функции высшего порядка (sorted, filter, map) позволяют писать гибкий и переиспользуемый код, отделяя логику обработки от структуры данных.

Паттерн «Стратегия» через callable-объекты обеспечивает взаимозаменяемость алгоритмов без изменения кода коллекции. Достаточно передать другую функцию или объект-стратегию.

Lambda-выражения удобны для простых одноразовых операций, делают код компактным. Однако для переиспользуемой логики предпочтительны именованные функции с docstring.

Цепочки вызовов (filter_by → sort_by → apply) делают код выразительным и читаемым, каждая операция четко обозначена.

Фабрики функций позволяют создавать параметризованные обработчики на лету, что удобно когда параметры фильтрации становятся известны только во время выполнения.

Все стратегии и функции вынесены в отдельный модуль strategies.py, коллекция расширена методами функционального стиля в collection.py, демонстрация разделена по уровням сложности в demo.py.
