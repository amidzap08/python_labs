### Реализованные интерфейсы:

**1. Printable** — строковое представление объекта
- `to_string() -> str`

**2. Comparable** — сравнение объектов
- `compare_to(other) -> int` (-1/0/1)

**3. Identifiable** — уникальный идентификатор
- `get_id() -> str`

**4. Validatable** — валидация состояния
- `is_valid() -> bool`
- `get_validation_errors() -> list`

### Реализация в классах:

**Team** — реализует все 4 интерфейса
- `to_string()`: `Team: Lakers (Pacific) | Points: 52 | Active: True`
- `compare_to()`: сравнивает по `performance_score()`
- `get_id()`: `lakers_pacific`
- `is_valid()`: проверяет название, дивизион, очки, мораль

**Player** — реализует 3 интерфейса (Printable, Comparable, Identifiable)
- `to_string()`: `Player: LeBron James | Pos: Forward | Team: Lakers | Rating: 96`
- `compare_to()`: сравнивает по `skill_rating`
- `get_id()`: `lebron_james_lakers`

**Match** — реализует 2 интерфейса (Printable, Comparable)
- `to_string()`: `Match: Lakers 112 - 108 Warriors (FINAL)`
- `compare_to()`: сравнивает по сумме голов

### Полиморфизм через интерфейсы:

**Интерфейс как тип:**
```python
def print_all(items: list[Printable]):
    for item in items:
        print(item.to_string())  # разное поведение для Team, Player, Match
```
