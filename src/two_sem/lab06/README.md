## 1. Типизированная коллекция TypedCollection

Коллекция реализована как обобщённый (generic) класс с параметром типа T. Обеспечивает типобезопасность на этапе статического анализа и поддерживает функции высшего порядка для обработки данных.

### Параметры типа:

**`T = TypeVar('T')`** — основной параметр типа для элементов коллекции

**`R = TypeVar('R')`** — параметр типа для результатов преобразования в методе map()

### Методы (Задание на 3):

**`add(item: T) -> None`** — добавление элемента типа T в коллекцию. Принимает объект, соответствующий параметру типа коллекции.

**`remove(item: T) -> bool`** — удаление элемента из коллекции. Возвращает True при успешном удалении.

**`get_all() -> List[T]`** — возвращает копию списка всех элементов коллекции.

**`__len__() -> int`** — количество элементов в коллекции.

**`__iter__() -> Iterator[T]`** — итератор для обхода элементов.

**`__contains__(item: T) -> bool`** — проверка наличия элемента в коллекции.

**`clear() -> None`** — очистка коллекции.

**`is_empty() -> bool`** — проверка на пустоту.

### Методы высшего порядка (Задание на 4):

**`find(predicate: Callable[[T], bool]) -> Optional[T]`** — поиск первого элемента, удовлетворяющего условию. Принимает функцию-предикат, возвращает найденный элемент или None.

**`filter(predicate: Callable[[T], bool]) -> List[T]`** — фильтрация элементов по условию. Принимает функцию-предикат, возвращает список подходящих элементов.

**`map(transform: Callable[[T], R]) -> List[R]`** — применение функции преобразования ко всем элементам. Тип результата R может отличаться от типа элементов T.

### Поддержка протоколов (Задание на 5):

Коллекция может быть параметризована с ограничением по протоколу:

```python
teams: TypedCollection[D] = TypedCollection()  # D bound=Displayable
teams: TypedCollection[S] = TypedCollection()  # S bound=Scorable
```
## 2. Протоколы структурной типизации
Реализовано 2 протокола, определяющих контракты через наличие методов без требования наследования:

Displayable (Protocol)
Определяет контракт для объектов, которые можно отобразить. Требует метод:

display() -> str — возвращает строковое представление объекта.

Класс Team соответствует протоколу при наличии метода display(). Наследование от Displayable не требуется.

Scorable (Protocol)
Определяет контракт для объектов, которые можно оценить. Требует метод:

score() -> float — возвращает числовую оценку объекта.

Класс Team соответствует протоколу при наличии метода score(). Наследование от Scorable не требуется.

TypeVar с ограничениями:
D = TypeVar('D', bound=Displayable) — принимает только объекты с методом display()

S = TypeVar('S', bound=Scorable) — принимает только объекты с методом score()

## 3. Аннотации типов
Все методы коллекции полностью аннотированы:

```python
def add(self, item: T) -> None:
    self._items.append(item)

def find(self, predicate: Callable[[T], bool]) -> Optional[T]:
    for item in self._items:
        if predicate(item):
            return item
    return None

def map(self, transform: Callable[[T], R]) -> List[R]:
    return [transform(item) for item in self._items]
```
Аннотации проверяются статическим анализатором mypy и обеспечивают:

Контроль типов при добавлении элементов

Корректность возвращаемых значений

Безопасность вызова методов на элементах коллекции

## 4. Функции высшего порядка
find() — поиск элемента
Принимает функцию-условие, возвращает первый подходящий элемент или None:


## Поиск команды по имени
team = teams.find(lambda t: t.name == "Спартак")

## Поиск команды в elite дивизионе
team = teams.find(lambda t: t.division == "elite")
filter() — фильтрация
Принимает функцию-условие, возвращает список подходящих элементов:

## Активные команды
active = teams.filter(lambda t: t.is_active)

## Команды с высокими очками
strong = teams.filter(lambda t: t.total_points >= 100)
map() — преобразование
Принимает функцию преобразования, возвращает список результатов. Тип результата определяется второй TypeVar R:

## Извлечение имён (Team -> str)
names: list[str] = teams.map(lambda t: t.name)

## Извлечение очков (Team -> float)
scores: list[float] = teams.map(lambda t: t.total_points)

## Создание кортежей (Team -> tuple)
info: list[tuple[str, float]] = teams.map(lambda t: (t.name, t.total_points))

## 6. Выводы

**Generic-классы** через TypeVar и Generic обеспечивают типобезопасность коллекций на этапе статического анализа, предотвращая ошибки несоответствия типов до выполнения программы.

**Протоколы (typing.Protocol)** реализуют структурную типизацию — объект соответствует протоколу, если имеет нужные методы, независимо от наследования. Это делает код гибче по сравнению с ABC.

**Ограниченные TypeVar (bound=)** позволяют создавать коллекции, которые принимают только объекты с определёнными методами, при этом не требуя явного наследования от интерфейса.

**Функции высшего порядка (find, filter, map)** в сочетании с аннотациями типов обеспечивают безопасную обработку данных с проверкой типов на каждом этапе преобразования.

**Второй TypeVar R в методе map()** наглядно демонстрирует изменение типа при преобразовании, что видно как на уровне кода, так и при проверке mypy.

**Статический анализ типов (mypy)** позволяет обнаруживать ошибки типов до запуска программы, что особенно полезно при рефакторинге и работе с большими кодовыми базами.

**Lambda**-выражения в сочетании с типизированными функциями высшего порядка позволяют писать компактный и при этом типобезопасный код.

**Типизированная коллекция TypedCollection** повторяет интерфейс TeamCollection из ЛР-2, но добавляет информацию о типе хранимых элементов, что делает код самодокументированным и безопасным.
