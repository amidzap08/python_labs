# Лабораторная работа 3
## Вариант 7
### Цель работы:
Освоить механизм наследования классов, научиться строить иерархию объектов, понять разницу между базовым и производным классами, освоить переопределение методов и полиморфизм.

### Постановка задачи:
На основе класса Team из ЛР-1 построить иерархию спортивных команд с разными характеристиками и поведением.

### Реализованная иерархия:

 Team (базовый класс)

├── ProfessionalTeam (профессиональная команда)

├── AmateurTeam (любительская команда)

└── YouthTeam (молодежная команда, наследник AmateurTeam)
## 1. Базовый класс Team (из ЛР-1)
### Атрибуты:

_name - название команды

_avg_weight - средний вес

_avg_height - средний рост

_total_points - набранные очки

_is_active - статус активности

_division - дивизион

### Методы:

avg_bmi() - расчет ИМТ

performance_score() - расчет производительности

activate() / deactivate() - управление активностью

set_division() - изменение дивизиона

add_points() - добавление очков

## 2. Производный класс ProfessionalTeam
Новые атрибуты :

_budget - бюджет команды

_sponsor_tier - уровень спонсорства (Bronze/Silver/Gold/Platinum)

_league - лига

_contract_bonus - бонус по контракту

### Атрибуты класса:

min_budget = 100000 - минимальный бюджет

sponsorship_tiers - допустимые уровни спонсорства

Новые методы (1+):

calculate_roi() - расчет возврата инвестиций

get_sponsorship_level() - описание уровня спонсорства

calculate_team_value() - расчет стоимости команды

### Переопределенные методы:

__str__() - добавлена информация о лиге, бюджете и спонсорстве

performance_score() - учитывает бюджет и уровень спонсорства

## 3. Производный класс AmateurTeam
Новые атрибуты (2+):

_membership_fee - членский взнос

_volunteer_count - количество волонтеров

_practice_frequency - частота тренировок

_community_events - счетчик общественных мероприятий

### Атрибуты класса:

max_membership_fee = 1000 - максимальный взнос

volunteer_roles - роли волонтеров

Новые методы (1+):

calculate_monthly_income() - расчет месячного дохода

get_practice_schedule() - расписание тренировок

add_community_event() - добавление мероприятия

calculate_team_value() - расчет стоимости команды

### Переопределенные методы:

__str__() - добавлена информация о взносах, волонтерах и тренировках

performance_score() - учитывает волонтеров и мероприятия

## 4. Производный класс YouthTeam (наследник AmateurTeam)
### Новые атрибуты (2+):

_age_group - возрастная группа (U16, U18 и т.д.)

_school_program - участие в школьной программе

_scholarship_count - количество выданных стипендий

### Новые методы (1+):

award_scholarship() - выдача стипендии

### Переопределенные методы:

__str__() - добавлена информация о возрастной группе

calculate_team_value() - базовая стоимость × 1.3

## 5. Полиморфизм
Общий интерфейс:

calculate_team_value() - разная логика расчета в каждом классе

performance_score() - разные множители для каждого типа команд

### Полиморфизм без условий:

```python
for team in collection:
    value = team.calculate_team_value()  # разное поведение
```
## 6. Коллекция TeamCollection
### Методы:

add_team() - добавление команды

get_professional_teams() - фильтрация профессиональных команд

get_amateur_teams() - фильтрация любительских команд

get_youth_teams() - фильтрация молодежных команд

get_statistics() - статистика по типам команд
