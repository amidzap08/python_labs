# Лабораторная работа 1
## Вариант 7
### Цель работы:
Реализовать класс, демонстрирующий основные принципы объектно-ориентированного программирования в Python на примере спортсмена.

* Объект класса:
Я выбрала класс Athlete (Спортсмен), потому что спортсмены имеют множество характеристик (возраст, вес, вид спорта, личные рекорды), которые требуют валидации, а также различные состояния и поведение (тренировки, улучшение результатов, активность/неактивность).


* Реализованные элементы:
## 1. Класс Athlete
Защищенные атрибуты экземпляра (6 атрибутов):
_name - имя спортсмена

_age - возраст

_sport_type - вид спорта

_weight - вес

_personal_record - личный рекорд

_is_active - статус активности (активен/неактивен)

### Состояние экземпляра контролируем через @property:
* Геттеры (@property) - контролируем чтение всех атрибутов

* Сеттеры (@<attr>.setter) - контролируем изменение с проверками:

   name.setter - проверка длины имени (от 2 до 50 символов)

   age.setter - проверка возрастного диапазона (от 14 до 50 лет)

   sport_type.setter - проверка соответствия допустимым видам спорта

    weight.setter - проверка веса (положительное число, не более 200 кг)

    personal_record.setter - проверка рекорда (неотрицательное число)

## 2. Атрибуты класса:
    sport_type_list - список допустимых видов спорта

    min_age - минимальный допустимый возраст

    max_age - максимальный допустимый возраст

##  3. В файле validate.py вынесены все проверки:
Класс AthleteValidator содержит статические и классовые методы для проверки:

Проверка типов данных:

check_name - проверка, что имя является строкой

check_age - проверка, что возраст целое число

check_weight - проверка, что вес число

check_personal_record - проверка, что рекорд число

check_is_active - проверка, что статус булевое значение

Проверка диапазона:

check_age - возраст в пределах [14, 50]

check_weight - вес в пределах (0, 200]

check_personal_record - рекорд ≥ 0

check_training_hours - часы тренировки в пределах (0, 8]

Проверка допустимых значений (из списка):

check_sport_type - вид спорта должен быть из списка: ["Футбол", "Баскетбол", "Плавание", "Легкая атлетика", "Теннис", "Бокс"]

Комплексная проверка:

check_all - проверка всех полей при создании объекта

## 4. Магические методы:
__str__ - красивый вывод для пользователя:


Спортсмен: Иван Петров | Возраст: 25 | Вид спорта: Футбол | Вес: 75.5 кг | Рекорд: 12.50 | Статус: активен

__repr__ - вывод для разработчика (отладочная информация):


Athlete(name='Иван Петров', age=25, sport_type='Футбол', weight=75.5, personal_record=12.5)
__eq__ - сравнение спортсменов по имени и виду спорта:

athlete1 == athlete2  # True если одинаковые имя и вид спорта
## 5. Бизнес-методы:
update_record(new_record) - обновление личного рекорда с проверкой улучшения

train(hours) - проведение тренировки с расчетом сожженных калорий

get_bmi() - расчет индекса массы тела

get_age_category() - определение возрастной категории (Юниор/Взрослый/Ветеран)

## 6. Логическое состояние:
deactivate() - деактивация спортсмена (снятие с соревнований)

activate() - активация спортсмена (возврат в спорт)

is_active - свойство для проверки текущего статуса

## 7. Валидация бизнес-логики:
Нельзя тренировать неактивного спортсмена

Нельзя обновить рекорд неактивного спортсмена

Для юных спортсменов (до 18 лет) рекомендуется тренироваться не более 4 часов

При деактивации проверяется, что спортсмен еще не деактивирован

При активации проверяется, что спортсмен не был уже активен
