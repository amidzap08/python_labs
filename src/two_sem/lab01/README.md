# Лабораторная работа 1
## Вариант 7
### Цель работы:
Реализовать класс, демонстрирующий основные принципы объектно-ориентированного программирования в Python на примере спортсмена.
Объект класса:
Я выбрала класс Athlete (Спортсмен), потому что спортсмены имеют множество характеристик (имя, вес, рост, личные рекорды), которые требуют валидации, а также различные состояния и поведение (активность/неактивность, уровень подготовки, здоровье, мораль).

Реализованные элементы:
1. Класс Athlete
Защищенные атрибуты экземпляра (8 атрибутов):

_name - имя спортсмена

_weight - вес (кг)

_height - рост (см)

_personal_record - личный рекорд (очки)

_is_active - статус активности (активен/неактивен)

_health_status - состояние здоровья (healthy/injured/recovering)

_training_level - уровень подготовки (beginner/intermediate/advanced/professional)

_morale - моральный дух (от 0 до 10)

2. Состояние экземпляра контролируем через @property:
Геттеры (@property) - контролируем чтение всех атрибутов

Сеттеры (@<attr>.setter) - контролируем изменение с проверками:

weight.setter - проверка веса (положительное число, от 20 до 300 кг)

height.setter - проверка роста (положительное число, от 50 до 280 см)

personal_record.setter - проверка рекорда (неотрицательное число, не более 500)

morale.setter - проверка морали (целое число от 0 до 10)

3. Атрибуты класса:
total_athletes - счетчик созданных спортсменов

sport_type - вид спорта (по умолчанию "running")

default_training_level - уровень подготовки по умолчанию ("beginner")

max_morale / min_morale - границы морали (10 и 0)

weight_unit / height_unit / record_unit - единицы измерения ("kg", "cm", "kg")

4. Ошибки валидации при создании объекта (через validate.py)
Тип имени — передать число вместо строки — имя должно быть строкой

Пустое имя — передать пустую строку — имя не может быть пустым

Длина имени — передать имя из 1 символа — имя должно содержать минимум 2 символа

Длина имени — передать имя длиннее 50 символов — имя не должно превышать 50 символов

Тип веса — передать строку — вес должен быть числом

Отрицательный вес — передать отрицательное число — вес должен быть положительным числом

Вес слишком маленький — передать вес менее 20 кг — вес не может быть меньше 20 кг

Вес слишком большой — передать вес более 300 кг — вес не может превышать 300 кг

Тип роста — передать строку — рост должен быть числом

Отрицательный рост — передать отрицательное число — рост должен быть положительным числом

Рост слишком маленький — передать рост менее 50 см — рост не может быть меньше 50 см

Рост слишком большой — передать рост более 280 см — рост не может превышать 280 см

Тип рекорда — передать строку — рекорд должен быть числом

Отрицательный рекорд — передать отрицательное число — рекорд не может быть отрицательным

Рекорд слишком большой — передать рекорд более 500 — рекорд не может превышать 500

Тип морали — передать строку — мораль должна быть целым числом

Мораль меньше минимума — передать значение менее 0 — мораль должна быть от 0 до 10

Мораль больше максимума — передать значение более 10 — мораль должна быть от 0 до 10

Некорректный уровень подготовки — передать уровень не из допустимого списка — уровень подготовки должен быть одним из: beginner, intermediate, advanced, professional

Тип уровня подготовки — передать число — уровень подготовки должен быть строкой

Некорректный статус здоровья — передать статус не из допустимого списка — статус здоровья должен быть одним из: healthy, injured, recovering

Тип статуса здоровья — передать число — статус здоровья должен быть строкой

Тип активности — передать строку — статус активности должен быть булевым значением (присваивается автоматически)

5. Магические методы:
__str__ - красивый вывод для пользователя:

text
Спортсмен: Иван Петров, вес: 75.5 kg, рост: 180 cm, рекорд: 120.0 kg, ИМТ: 23.3, здоровье: healthy, уровень: intermediate, мораль: 8/10, статус: активен
__repr__ - вывод для разработчика (отладочная информация):

text
Athlete(name='Иван Петров', weight=75.5, height=180, record=120.0)
__eq__ - сравнение спортсменов по имени, весу, росту и рекорду:

python
athlete1 == athlete2  # True если одинаковые имя, вес, рост и рекорд
6. Бизнес-методы:
bmi() - расчет индекса массы тела

performance_score() - расчет производительности с учетом уровня подготовки, здоровья и морали

set_record(new_record) - обновление личного рекорда

set_training_level(level) - изменение уровня подготовки

7. Логическое состояние:
deactivate() - деактивация спортсмена (снятие с соревнований)

activate() - активация спортсмена (возврат в спорт)

is_active - свойство для проверки текущего статуса

Управление здоровьем:

injure() - получение травмы

recover() - начало восстановления

heal() - полное излечение

8. Валидация бизнес-логики:
Нельзя установить рекорд неактивному спортсмену

Нельзя установить рекорд при травме (injured)

При восстановлении (recovering) установка рекорда возможна, но выводится предупреждение

Нельзя изменить уровень подготовки неактивного спортсмена

Рекорд не может превышать максимальное значение для текущего уровня подготовки:

beginner: 100

intermediate: 200

advanced: 350

professional: 500
