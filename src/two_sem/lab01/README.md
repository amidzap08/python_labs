# python_labs
# Лабораторная работа 1
## Вариант 7
### Цель работы:
Реализовать класс, демонстрирующий основные принципы объектно-ориентированного программирования в Python на примере спортивной команды.

Объект класса:
Я выбрала класс Team (Спортивная команда), потому что команды имеют множество характеристик (название, средний вес, средний рост, набранные очки), которые требуют валидации, а также различные состояния и поведение (активность/неактивность, дивизион, здоровье, мораль).

Реализованные элементы:

## 1. Класс Team
Защищенные атрибуты экземпляра (8 атрибутов):

**_name** - название команды

**_avg_weight** - средний вес команды (кг)

**_avg_height** - средний рост команды (см)

**_total_points** - набранные очки

**_is_active** - статус активности (активна/неактивна)

**_health_status** - состояние здоровья команды (healthy/injured/recovering)

**_division** - дивизион команды (beginner/intermediate/professional/elite)

**_morale** - моральный дух команды (от 0 до 10)

## 2. Состояние экземпляра контролируем через @property:
**Геттеры (@property)** - контролируем чтение всех атрибутов

**Сеттеры (@<attr>.setter)** - контролируем изменение с проверками:

**avg_weight.setter** - проверка среднего веса (положительное число, от 20 до 300 кг)

**avg_height.setter** - проверка среднего роста (положительное число, от 50 до 280 см)

**total_points.setter** - проверка очков (неотрицательное число, не более 500)

**morale.setter** - проверка морали (целое число от 0 до 10)

## 3. Атрибуты класса:
**total_teams** - счетчик созданных команд

**sport_type** - вид спорта (по умолчанию "running")

**default_division** - дивизион по умолчанию ("beginner")

**max_morale / min_morale** - границы морали (10 и 0)

**weight_unit / height_unit / record_unit** - единицы измерения ("kg", "cm", "points")

## 4. Ошибки валидации при создании объекта (через validate.py)
**Тип имени** — передать число вместо строки — имя должно быть строкой

**Пустое имя** — передать пустую строку — имя не может быть пустым

**Длина имени** — передать имя из 1 символа — имя должно содержать минимум 2 символа

**Длина имени** — передать имя длиннее 50 символов — имя не должно превышать 50 символов

**Тип веса** — передать строку — вес должен быть числом

**Отрицательный вес** — передать отрицательное число — вес должен быть положительным числом

**Вес слишком маленький** — передать вес менее 20 кг — вес не может быть меньше 20 кг

**Вес слишком большой** — передать вес более 300 кг — вес не может превышать 300 кг

**Тип роста** — передать строку — рост должен быть числом

**Отрицательный рост** — передать отрицательное число — рост должен быть положительным числом

**Рост слишком маленький** — передать рост менее 50 см — рост не может быть меньше 50 см

**Рост слишком большой** — передать рост более 280 см — рост не может превышать 280 см

**Тип очков** — передать строку — очки должны быть числом

**Отрицательные очки** — передать отрицательное число — очки не могут быть отрицательными

**Очки слишком большие** — передать очки более 500 — очки не могут превышать 500

**Тип морали** — передать строку — мораль должна быть целым числом

**Мораль меньше минимума** — передать значение менее 0 — мораль должна быть от 0 до 10

**Мораль больше максимума** — передать значение более 10 — мораль должна быть от 0 до 10

**Некорректный дивизион** — передать дивизион не из допустимого списка — дивизион должен быть одним из: beginner, intermediate, professional, elite

**Тип дивизиона** — передать число — дивизион должен быть строкой

**Некорректный статус здоровья** — передать статус не из допустимого списка — статус здоровья должен быть одним из: healthy, injured, recovering

**Тип статуса здоровья** — передать число — статус здоровья должен быть строкой

**Тип активности** — передать строку — статус активности должен быть булевым значением (присваивается автоматически)

## 5. Магические методы:
### __str__ - красивый вывод для пользователя:

Команда: Спартак, средний вес: 70kg, средний рост: 175cm, очки: 100points, средний ИМТ: 22.86, здоровье: healthy, дивизион: beginner, уровень мотивации: 10/10, статус: активна

### __repr__ - вывод для разработчика (отладочная информация):

Team(name='Спартак', avg_weight=70, avg_height=175, total_points=100)

### __eq__ - сравнение команд по названию, среднему весу, среднему росту и очкам:

team1 == team2  # True если одинаковые название, средний вес, средний рост и очки

## 6. Бизнес-методы:
**avg_bmi()** - расчет среднего индекса массы тела команды

**performance_score()** - расчет производительности с учетом дивизиона, здоровья и морали

**set_points(new_points)** - обновление набранных очков

**set_division(division)** - изменение дивизиона команды

## 7. Логическое состояние:
**deactivate()** - деактивация команды (снятие с соревнований)

**activate()** - активация команды (возврат в спорт)

**is_active** - свойство для проверки текущего статуса

### Управление здоровьем:

**injure()** - получение травм в команде

**recover()** - начало восстановления

**heal()** - полное излечение

## 8. Валидация бизнес-логики:
**Нельзя изменить очки неактивной команды**

**Нельзя изменить очки при травмах в команде (injured)**

**При восстановлении (recovering) изменение очков возможно, но выводится предупреждение**

**Нельзя изменить дивизион неактивной команды**

**Очки не могут превышать максимальное значение для текущего дивизиона:**
beginner: 100

intermediate: 200

professional: 350

elite: 500
